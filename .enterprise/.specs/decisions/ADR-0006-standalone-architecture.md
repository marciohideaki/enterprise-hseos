## Context

HSEOS v1.x is approximately 92% standalone but carries three load-bearing dependencies on the host machine that break the "any Agent Code, any machine, no external assets" promise:

1. `swarm.agent.yaml` declares `~/.claude/skills/dev-squad/SKILL.md` as a `mandatory_read`. The file lives outside the repo, so a clean clone cannot bootstrap SWARM.
2. The Axon MCP server is configured only in `~/.claude/.mcp.json`. The project has no `.mcp.json`, and `allowedMcpServers: []` blocks global MCP usage.
3. The `second-brain` skill is loaded by CIPHER, GHOST, BLITZ, and ORBIT. Documentation presumes `second_brain.enabled: true` and a vault at `/opt/hideakisolutions/second-brain/`. A graceful fallback exists but is poorly signalled.

The 2026 ecosystem (MCP 2025-11-25 with Tasks/streamable HTTP, A2A absorbing ACP under LF AAIF, AGENTS.md adopted by 18+ tools across 60k+ repos, SKILL.md open standard with 32+ adoptions since dec/2025, NIST CAISI AI Agent Standards Initiative) has set new baselines that HSEOS is not yet meeting. Without standalone, HSEOS cannot ship as a redistributable framework, cannot index in Smithery / agentskills.io / AAIF members directory, and cannot be adopted by teams that do not run the same workstation as the original author.

---

## Decision

We will reorganize HSEOS as a fully standalone, vendor-neutral framework under the following invariants:

**P1 — Single Source of Truth.** `.agents/`, `.hseos/`, and `.enterprise/` are the only authoritative inputs. Nothing outside the repository is load-bearing at runtime.

**P2 — Adapters are derived.** Every adapter directory (`.claude/`, `.codex/`, `.cursor/`, `.continue/`, `.aider/`, `.cline/`) is output of the compiler. Generated files carry a sentinel header (`# DO NOT EDIT — generated by hseos agent-core compile`). User-owned files (`lessons.md`, `settings.local.json` user permissions) are preserved through non-destructive merge.

**P3 — Hash-pinned integrity.** `manifest.yaml` v2 stores `sha256` for each compiled asset under `.agents/.signatures/`. Drift detection runs on `SessionStart` and in CI.

**P4 — Vendor parity.** Capabilities are expressed in vocabulary neutral to any vendor; each adapter maps to the specific syntax of its target. When a capability is unsupported by an adapter, the fallback is documented in `.agents/instructions/CAPABILITY-MATRIX.md`.

**P5 — Zero global path.** No `~/.claude`, `/opt/...`, `$HOME/...` reference appears in any runtime asset (skill, agent, workflow, hook). Global paths are permitted only inside opt-in bootstrap flows that the user must explicitly enable.

**P6 — Graceful degradation.** Every external integration (Axon, second-brain vault, enterprise MCPs) has a documented null-mode fallback. HSEOS must function with zero MCPs and zero plugins installed.

**P7 — Tier policy machine-readable.** Tier 1 (always-load) versus Tier 2 (trigger-load) is declared in SKILL.md frontmatter and enforced by the skill loader; adapter-specific overrides live in the same frontmatter block.

The migration is delivered in ten waves (W0..W9) culminating in a versioned breaking release `v2.0.0`.

---

## Consequences

### Positive
- Clean clone runs anywhere — Linux, macOS, Windows (WSL), Docker, CI — with zero host-state dependencies.
- HSEOS becomes a redistributable framework: publishable on npm, Smithery, agentskills.io, AAIF members directory.
- Drift between source and compiled adapters is mechanically detectable via hash chain.
- Cross-vendor adoption: any of the 18+ AGENTS.md-aware tools and any of the major MCP-enabled coding assistants can consume HSEOS without modification.

### Negative / Trade-offs
- Breaking change: existing installations require a v1→v2 migration path documented in `MIGRATION-GUIDE.md`.
- Compiler complexity grows: 6+ adapter implementations to maintain.
- The vault integration (second-brain) becomes opt-in and ships disabled by default; users who relied on `enabled: true` must explicitly flip the flag.

### Risks
- Adapter parity gaps may surface where a vendor lacks a specific hook/skill primitive — mitigated by `CAPABILITY-MATRIX.md` and documented fallbacks per P4.
- Generated-file drift if users hand-edit adapter outputs — mitigated by sentinel headers and `hseos verify` running in CI.
- Wave 1 (decoupling) and Wave 2 (compiler refactor) carry the highest blast radius — mitigated by `pre-w<N>` rollback tags and clean-env smoke tests in Docker before merging each wave.

## Alternatives Considered

| Alternative | Why Rejected |
|---|---|
| Keep `~/.claude/skills/dev-squad/` as authoritative, sync to repo periodically | Manual sync cycle drifted silently (Wave 5a "State emission contract" was already lost between global and repo). Single SoT eliminates an entire class of drift. |
| Gitignore adapter directories entirely | Loses `lessons.md` and project-local permissions in `settings.local.json`. Sentinel-header merge preserves these without sacrificing portability. |
| Author one adapter (Anthropic platform) and let others fork | Contradicts the framework's positioning ("any Agent Code"). Forfeits 5 of 6 candidate vendors and the AAIF cross-vendor positioning. |
| Single bundled MCP set with no tiering | Forces Axon and enterprise-only MCPs (postgres, kubernetes, sentry) onto every install. Three-tier bundle (core/extended/enterprise) preserves graceful degradation per P6. |
