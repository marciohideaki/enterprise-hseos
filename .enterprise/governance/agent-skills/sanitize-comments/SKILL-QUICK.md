# Sanitize Comments — Quick Checklist

> Load when reviewing or cleaning source code comments before commit.
> Removes references to project management methodology, AI tooling, and process artifacts.
> NEVER modifies executable code — comments only.

---

## What to Remove

- [ ] References to FR / NFR / AC / Story / Epic / Sprint / Backlog / Roadmap
- [ ] "Conforme requisito X", "Atende ao FR-…", "Relacionado à Story …"
- [ ] References to Jira / Azure DevOps / tickets / work items
- [ ] References to AI tools: Claude, HSEOS, Copilot, ChatGPT, Cursor, Gemini, Codex, OpenAI, Anthropic, LLM
- [ ] "Generated by AI", "AI-assisted", "suggested by AI", "via Claude"
- [ ] Co-authorship credits in file headers or inline comments

## What to Preserve

- [ ] Comments explaining **what the code does** (behavior, algorithm)
- [ ] Comments explaining **why** a decision was made (architectural rationale)
- [ ] Invariants, preconditions, postconditions
- [ ] TODO/FIXME with technical context (no methodology refs)
- [ ] Public API documentation (XMLDoc, TSDoc, JavaDoc, DartDoc, GoDoc)

## Hard Constraints

- [ ] Zero changes to executable code — logic, names, types unchanged
- [ ] No new comments added
- [ ] No rewriting of surviving comments (preserve exact wording)
- [ ] No summarization or paraphrasing

## Word-Boundary Rule (same as commit-hygiene)

Match AI terms as **whole words only**. "domain" ≠ "AI". "email" ≠ "AI".
Never flag a comment for containing "ai" as a substring.
