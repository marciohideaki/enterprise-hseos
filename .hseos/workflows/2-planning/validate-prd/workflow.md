# Validate PRD

## Intent
Check a PRD for completeness, lean scope, and readiness for architecture and story slicing.

## Owner
VECTOR

## Phases
1. Check goals, non-goals, personas, requirements, and acceptance criteria.
2. Identify scope creep, contradictions, and missing constraints.
3. Return a pass/block verdict with required fixes.

## Gates
- Block when acceptance criteria are not executable.
