## 1. O que o ciclo entregou

### 1.1 `audit` deixou de ser um clone do `verify`

Gap V3 (aberto desde o ciclo 01): `agent-core audit` emitia exatamente o mesmo output do `verify` — detectava drift sem dizer QUAL lado moveu. Agora o audit **triangula cada artefato pinado**: recomputa o hash a partir da fonte `.enterprise` (sem escrever nada), compara com o compilado em disco e com o pin do manifest, e emite **veredito direcional com remédio específico**:

| Estado (fonte × compilado × manifest) | Veredito | Remédio |
|---|---|---|
| todos iguais | `in sync` | — |
| fonte difere; compilado==manifest | `source edited since last compile` | compile propaga |
| compilado difere; fonte==manifest | `compiled output hand-edited` | compile descarta a edição (fonte é soberana); portar à fonte se intencional |
| fonte==compilado; ambos≠manifest | `manifest stale` | compile refresca o pin |
| os três diferem | `diverged independently` | reconciliar manualmente e compilar |

- **Skills**: fresh = re-execução do `normalizeSkill` sobre a fonte (mesma transformação do compiler).
- **Handlers**: fresh = fonte CRLF-normalizada (mesma normalização do sync).
- **Agents**: 2 pontos por construção (o `.agent.yaml` É a fonte) — veredito explícito disso.
- **Projetos instalados sem fonte `.enterprise`**: degradação explícita para comparação de 2 pontos ("source unavailable — direction unknown").

Total: **114 checks** no audit (88 triangulares + 26 de adapter-drift), agora com semântica distinta do verify.

### 1.2 Validação comportamental (não só código)

Exercitado ao vivo nos três estados: repo limpo → `in sync`; edição manual do compilado → `compiled output hand-edited`; estado misto real → `diverged independently` (veredito correto para o estado que meu próprio cleanup imperfeito criou — o audit explicou o que aconteceu melhor que o operador). Testes permanentes: +4 asserções em `test-agent-core-verify-audit-doctor.js` (in-sync, hand-edited, source-edited, fallback sem fonte) — **20/20**.
