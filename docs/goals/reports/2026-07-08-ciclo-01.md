## 1. Relatório Arquitetural

O mapa arquitetural detalhado vive em `docs/pt-br/especificacao-hseos.md` (gerado em 2026-07-08 e **corrigido por este ciclo** — ver §2, hipóteses falsificadas). Fatos verificados neste ciclo direto do código:

- **Fluxo fonte→compilado funciona de ponta a ponta**: `.enterprise/governance/agent-skills/` → `agent-core compile` → `.agents/` (manifest hash-pinned) → adapters (`.claude/hooks.json`, `.codex/`). O pipeline modular (`sources/ → adapters/ → manifest/`) compila 49 skills, 28 hooks, 9 commands, 16 agents, 4 plugins, 8 servers MCP.
- **Validação independente do reparo de 2026-07-08**: o `.claude/hooks.json` regenerado pelo compiler ficou **byte-idêntico** à resolução manual do conflito — a escolha do reparo era exatamente o que o registry produz.
- **Auto-referência no subsistema de hooks (smell arquitetural)**: `registry.yaml` é simultaneamente *input* e *output* do compiler, vivendo na árvore de saída `.agents/` — único artefato que viola o fluxo unidirecional `.enterprise → .agents`. Consequência prática: comentários/documentação no registry são destruídos a cada compile (mitigado neste ciclo com cabeçalho autodocumentado emitido pelo próprio compiler).
- **Enforcement em 4 camadas confirmado** (hooks tempo-real → husky → CI → branch protection), mas com furo: nenhum dos dois `swarm-gate.sh` tem cobertura de teste (hook *bloqueante* de governança sem testes).
- **Estado**: SQLite (`as_*` + FTS5) com runner de migrations correto (`PRAGMA user_version`, idempotente) — o defeito encontrado estava num teste órfão que reimplementava o replay, não na produção.

13. **[installer]**: 3 algoritmos de idempotência distintos (hash+manifest / mtime / hash-fallback-mtime); ~metade de `core/manifest.js` sem call sites; `deepMerge` triplicado; RTK install patcheia `~/.claude/settings.json` GLOBAL a partir de install de projeto (tensão com P5). `tools/build-docs.mjs` quebrado (importa `website/` inexistente).
14. **[governança]**: `standards-adoption-metrics.md` é 100% aspiracional (nenhum mecanismo real); sistema de exceções nunca usado (0 EXC-XXXX); `replay-mode.active` é template vazio que tecnicamente ativa Replay Mode desde dez/2025; `validate-skills.sh` não é invocado por nada; ADR-0015 exige seção `## Authority` no dev-squad SKILL.md — nunca adicionada; sem shellcheck para ~20 scripts .sh.
15. **[worktree/qualidade]**: `.worktrees/stacked-branch-policy` órfão (remove não rodou); Bug #2 do WAVE-1 segue real (`gate_code` decide rodar por staged files mas linta o repo inteiro); manifest do compiler declara plataformas sem emissor real (cursor/continue/aider/cline) e `.claude/commands` que o pipeline não escreve.
16. **[MCP protocolo]**: os 4 servers implementam subset mínimo hand-rolled (protocolVersion `2024-11-05` hardcoded; sem resources/prompts) — avaliar migração para o SDK oficial ou ao menos version bump ao alinhar com o RC 2026-07-28; axon-bridge spawna processo por chamada sem handshake `initialize`, timeout 5s hardcoded.

## 4. Relatório de Aderência

- **P1–P7 (ADR-0006)**: P3 (integridade hash) estava **quebrado** no início do ciclo (42 mismatches) — restaurado. P5 (zero global path): o `scripts/governance/swarm-gate.sh` referencia conceitualmente o hook global `~/.claude` como "gate global" — a migração Wave-4 que resolve isso está escrita porém não fiada (gap #1).
- **Instruction cascade / stop-the-line**: respeitado neste ciclo — a consolidação do swarm-gate foi *preparada* e não executada por exigir decisão humana (§11).
- **Idempotência/determinismo**: dois defeitos de determinismo corrigidos (lint sobre temp files; migrations replay em teste). `verify` agora é reprodutível e verde.
- **Convenções próprias**: "1 commit = 1 task" não aplicável (ciclo não comita — decisão humana pendente sobre o working tree); quality gates exercitados via suíte.

## 5. Relatório de Benchmark (estado da arte, jul/2026)

- **Agent Skills / SKILL.md**: padrão aberto (Anthropic, dez/2025) sob governança AAIF/Linux Foundation; 32+ ferramentas (Claude Code, Codex, VS Code, Gemini CLI, Goose, etc.). O modelo HSEOS (SKILL.md + frontmatter `name`/`description` + tiers) é **compatível e aditivo** (`trigger`/`skip`/`tier` são extensões). O `suggest-skill.sh` já se declara compatível com agentskills.io. Posição: alinhado; oportunidade de publicar conformidade explícita.
- **MCP**: spec estável 2025-11-25; **RC 2026-07-28** (20 dias) traz stateless core, response caching (`ttlMs`/`cacheScope`), extensions framework, Tasks redesenhado, W3C Trace Context. Servers nativos HSEOS são implementações manuais JSON-RPC/stdio — validar versão de protocolo declarada e planejar upgrade pós-GA. FALTA: nenhum plano registrado para o RC.
- **Camadas do ecossistema** (MCP=tools, A2A=agent-to-agent, Skills=conhecimento procedural, AGENTS.md=config de projeto): a decomposição HSEOS espelha 1:1 (bundles MCP, skill `inter-agent-comms`, skills registry, cascade AGENTS.md) — arquitetura na direção do consenso de mercado.
